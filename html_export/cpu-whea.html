WHEA_UNCORRECTABLE_ERROR

BSOD chybový kód 0x0000124 – Windows zachytil Machine Check Exception přímo z CPU nebo chipset. Nastává při hardwarové nestabilitě: přetaktování mimo limity, podnapájení jádra nebo vadná paměť RAM způsobující data corruption.

Reseni krok za krokem

  1. Zakažte veškerý overclock v BIOSu – načtěte Load Optimized Defaults a otestujte stabilitu.
  2. Pokud XMP/EXPO profil RAM je aktivní, zkuste RAM na výchozí JEDEC frekvenci (2133/3200 MHz) a sledujte, zda BSOD zmizí.
  3. Spusťte paměťový test: Přes Windows Security → Diagnostika paměti nebo lépe bootovatelné MemTest86 (přes USB, alespoň 2 průchody).
  4. Zkontrolujte Event Viewer: Win+X → Event Viewer → Windows Logs → System, filtrujte zdroj WHEA-Logger – záznamy odhalí přesný vadný komponent.
  5. Při Intel 13/14. gen CPU: Nainstalujte nejnovější BIOS s opravou degradace (Intel Baseline Profile) a omezte PL1/PL2 na spec hodnoty.

Technicke detaily

WHEA (Windows Hardware Error Architecture) je low-level framework zachytávající Machine Check Exceptions (MCE) přímo z CPU. BSOD kód 0x0000124 je vyvolán, když CPU nahlásí nekorigovatelnou chybu přes MCE mechanismus (MSR registr IA32_MCG_STATUS). Nejčastější zdroje: (1) L1/L2/L3 cache parity error – nastává při undervoltu procesoru pod stability hranici nebo při přetaktování nad maximální frekvenci. (2) Interní UPI/Infinity Fabric link error u víceprocesorových systémů. (3) Vadná paměť způsobující data corruption při přístupu z CPU prefetcheru. (4) Koroze CPU kontaktů vlivem vlhkosti. Pokročilá diagnostika: v Event Vieweru → Windows Logs → System filtrujte na zdroj 'WHEA-Logger' – záznamy obsahují přesný MCi_STATUS registr identifikující postiženou část CPU (FPU, Load/Store, Instruction Fetch Unit).

English

BSOD error code 0x0000124 – Windows caught a Machine Check Exception directly from the CPU or chipset. Occurs during hardware instability: overclocking beyond limits, CPU undervolt below threshold, or faulty RAM causing data corruption.

  1. Disable all overclocking in BIOS – load Optimized Defaults and test for stability.
  2. If XMP/EXPO RAM profile is active, try RAM at stock JEDEC frequency (2133/3200 MHz) to isolate the issue.
  3. Run a memory test: Use Windows Memory Diagnostic or bootable MemTest86 (USB, at least 2 passes).
  4. Check Event Viewer: Win+X → Event Viewer → Windows Logs → System, filter source WHEA-Logger – logs reveal the exact failing component.
  5. Intel 13/14th gen CPU: Flash the latest BIOS with the degradation fix (Intel Baseline Profile) and limit PL1/PL2 to spec values.
