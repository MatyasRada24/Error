Code 43

Windows zastavil grafickou kartu (chybová zpráva ve Správci zařízení). Ovladač GPU ohlásil kernelu neobnovitelnou chybu přes TDR mechanismus – karta přestala komunikovat s OS. Nejčastěji jde o poškozený ovladač, vadnou VRAM nebo fyzicky poškozenou kartu.

Reseni krok za krokem

  1. Čistá reinstalace ovladače: Stáhněte DDU (Display Driver Uninstaller), spusťte Windows v Safe Mode, odinstalujte vše a nainstalujte nejnovější ovladač z webu NVIDIA/AMD.
  2. Přesuňte kartu do jiného PCIe slotu – testujete tak, zda nejde o vadný slot na základní desce.
  3. Zkontrolujte napájení: Ujistěte se, že jsou zapojeny všechny PCIe napájecí konektory (8-pin / 16-pin) přímo z PSU (ne přes rozbočovač/adaptér).
  4. Ověřte teplotu GPU v GPU-Z nebo HWInfo64 – přehřátí nad 90 °C způsobuje TDR resety. Vyčistěte prach a vyměňte termální pastu.
  5. Testujte GPU v jiném PC – pokud chyba přetrvá, je potvrzeno fyzické poškození hardwaru (BGA spoje, VRAM).

Technicke detaily

Kód 43 je generická chyba na úrovni kernel mód ovladače (TDR/Device timeout). Když Windows obdrží od GPU signál o chybě, Kernel Mode Driver Framework (KMDF) okamžitě zastaví zařízení a zaznamená chybu do Event Vieweru. Nejčastější příčiny: (1) Poškozené BGA spoje pod GPU jádrem vlivem teplotního cyklu – dochází k mikroprasklinám, které vedou k přerušovanému kontaktu. (2) Defektní paměťový čip VRAM (GDDR6/6X) způsobující nevalidní ECC opravy. (3) Korupce ovladačového souboru nvlddmkm.sys nebo amdkmdag.sys. (4) Nedostatek napájení – PCIe slot poskytuje max 75W, u karet nad 200W je nutné 8pin/16pin konektory. Diagnostický postup: spusťte GPU-Z, zkontrolujte Bus Interface (měl by být PCI-E x16 3.0/4.0), ověřte teplotu a napájecí limity. Poté spusťte DDU v Safe Mode, nainstalujte ovladač od nuly. Pokud karta stále generuje chybu 43 v jiném systému, je potvrzeno fyzické poškození HW.

English

Windows has stopped the GPU (shown in Device Manager). The GPU driver reported an unrecoverable error to the kernel via the TDR mechanism. Most commonly caused by a corrupted driver, faulty VRAM, or physically damaged card.

  1. Clean driver reinstall: Download DDU, boot into Safe Mode, uninstall all GPU drivers, then install the latest from NVIDIA/AMD website.
  2. Try a different PCIe slot – rules out a damaged slot on the motherboard.
  3. Check power connectors: Ensure all PCIe power cables (8-pin / 16-pin) are plugged directly from the PSU (no splitters).
  4. Monitor GPU temperature via GPU-Z or HWInfo64 – temps above 90°C cause TDR resets. Clean dust and replace thermal paste if needed.
  5. Test GPU in another PC – if the error persists, physical damage (BGA joints, VRAM) is confirmed.
